1. 兄弟字符串判断
题目：如果两个字符串的字符一样，但是顺序不一样，被认为是兄弟字符串，问如何在迅速匹配兄弟字符串（如，bad 和 adb 就是兄弟字符串）。
思路：判断各自素数乘积是否相等。
算法：素数乘积法
- 为每个字母分配一个唯一的素数
- 计算字符串中所有字符对应素数的乘积
- 如果两个字符串的乘积相等，则它们是兄弟字符串
时间复杂度：O(n)，空间复杂度：O(1)
适用场景：字符串相似度判断、字符集合比较、快速去重和分组、单词变位词检测

2. 集合合并
题目：给定一个字符串的集合，格式如：{aaa bbb ccc}， {bbb ddd}，{eee fff}，{ggg}，{ddd hhh}要求将其中交集不为空的集合合并，要求合并完成后的集合之间无交集，例如上例应输出{aaa bbb ccc ddd hhh}，{eee fff}， {ggg}。
算法：并查集（Union-Find）
- 为每个元素创建一个集合
- 遍历所有集合，将有交集的集合合并
- 最终输出所有合并后的集合
时间复杂度：O(nα(n))，空间复杂度：O(n)
适用场景：集合合并问题、连通分量查找、等价类划分

3. 矩阵旋转
题目：将一个二维N*N矩阵顺时针旋转90，例如a[2][2]={1,2,3,4}，旋转过后a[2][2]={4,1,3,2}。
算法：按层旋转法
- 从外到内逐层处理
- 每层元素按顺时针方向交换
- 每一层需要处理n-1-2*i次交换（i为层数）
时间复杂度：O(n²)，空间复杂度：O(1)
适用场景：图像处理、矩阵变换、游戏开发中的旋转效果

4. 数组乘积
题目：输入一个长度为n的整数数组 input，输出一个长度为n的整数数组result，满足result[i]=input 数组中除了input[i]之外所有数的乘积(假设不会溢出)。比如输入: input={2,3,4,5}，输出result={60,40,30,24}。
要求：具有线性复杂度，且不能使用除法运算符。
算法：左右乘积数组法
- 计算左侧乘积数组：left[i]表示i左侧所有元素的乘积
- 计算右侧乘积数组：right[i]表示i右侧所有元素的乘积
- result[i] = left[i] * right[i]
时间复杂度：O(n)，空间复杂度：O(1)（除输出数组外）
适用场景：数组元素乘积计算、数学计算中的前缀积和后缀积、不使用除法的情况下计算元素乘积

5. 寻找中间元素
题目：一个int数组，里面数据无任何限制，要求求出所有这样的数 a[i]，其左边的数都小于等于它，右边的数都大于等于它。
算法：双数组法
- 计算左侧最大值数组：leftMax[i]表示i左侧的最大值
- 计算右侧最小值数组：rightMin[i]表示i右侧的最小值
- 如果leftMax[i] <= nums[i] <= rightMin[i]，则nums[i]是中间元素
时间复杂度：O(n)，空间复杂度：O(n)
适用场景：寻找数组中的"枢纽"元素、数据流中的中位数查找、数组分区问题

6. 多数元素
题目：数组中有一个数字出现的次数超过了数组长度的一半，请找出这个数字。
算法：摩尔投票算法（Moore Voting Algorithm）
- 初始化候选元素和计数器
- 遍历数组进行投票：
  - 如果计数器为0，将当前元素设为候选元素
  - 如果当前元素等于候选元素，计数器加1，否则减1
- 验证候选元素是否是多数元素
时间复杂度：O(n)，空间复杂度：O(1)
适用场景：寻找数组中的多数元素、投票统计、众数查找

7. 只出现一次的数字
题目：一个整型数组里除了两个数字之外，其他的数字都出现了两次。请写程序找出这两个只出现一次的数字。要求时间复杂度是O(n)，空间复杂度是O(1)。
算法：位运算法
- 计算所有数字的异或结果，得到两个只出现一次数字的异或
- 找到异或结果中最低位的1，作为分组标准
- 根据分组标准将数组分为两组，分别异或得到两个只出现一次的数字
时间复杂度：O(n)，空间复杂度：O(1)
适用场景：寻找数组中唯一出现的元素、数据去重、位运算优化

9. 有序数组两数之和
题目：输入一个已经按升序排序过的数组和一个数字，在数组中查找两个数，使得它们的和正好是输入的那个数字。要求：时间复杂度是O(n)。如果有多对数字的和等于输入的数字，输出任意一对。
算法：双指针法
- 初始化左指针指向数组开始，右指针指向数组末尾
- 计算两指针指向元素的和：
  - 如果和等于目标值，返回这两个元素
  - 如果和小于目标值，左指针右移
  - 如果和大于目标值，右指针左移
时间复杂度：O(n)，空间复杂度：O(1)
适用场景：有序数组中的两数之和问题、查找满足条件的两个元素、范围搜索

10. 最小化和差
题目：有两个序列a,b，大小都为n,序列元素的值任意整数，无序；要求：通过交换a,b 中的元素，使 [序列a 元素的和] 与 [序列b 元素的和] 之间的差最小。
算法：动态规划法（背包问题）
- 计算两个数组的总和sum
- 计算目标值target = sum / 2
- 使用动态规划找出最接近target的可能和
- 计算最小差 = |sum - 2 * best_sum|
时间复杂度：O(n*sum)，空间复杂度：O(sum)
适用场景：资源分配问题、平衡问题、背包问题

11. 数组分块最大值
题目：一个整数数组a，长度为n，将其分为m份，使各份的和相等，求m的最大值。比如{3，2，4，3，6} 可以分成{3，2，4，3，6} m=1; {3,6}{2,4,3} m=2; {3,3}{2,4}{6} m=3 所以m的最大值为3。
算法：因数分解+回溯
- 计算数组总和sum
- 找出sum的所有因数，从大到小排序
- 对于每个因数k，尝试将数组分成k份，每份和为sum/k
- 使用回溯法判断是否可以成功分块
时间复杂度：O(n*2^n)，空间复杂度：O(n)
适用场景：数组分块问题、资源分配、均分问题

12. 有序数组找相同元素
题目：判断两个有序数组中是否存在相同的数字？给定两个排好的数组（规模长度为n）,怎样高效的判断这两个数组中存在相同的数字。给出O(n)算法。
算法：双指针法
- 初始化两个指针分别指向两个数组的开始
- 比较两个指针指向的元素：
  - 如果相等，返回存在
  - 如果第一个数组的元素小，移动第一个指针
  - 如果第二个数组的元素小，移动第二个指针
- 遍历结束后，返回不存在
时间复杂度：O(n)，空间复杂度：O(1)
适用场景：有序数组比较、集合交集判断、归并操作

28. 跳跃游戏
题目：给你一个非负整数数组 nums ，你最初位于数组的 第一个下标 。数组中的每个元素代表你在该位置可以跳跃的最大长度。判断你是否能够到达最后一个下标，如果可以，返回 true ；否则，返回 false 。
示例 1：输入：nums = [2,3,1,1,4]，输出：true（可以先跳 1 步，从下标 0 到达下标 1, 然后再从下标 1 跳 3 步到达最后一个下标）。

- 遍历数组，更新最远位置
- 如果最远位置大于等于数组长度-1，返回true
- 如果遍历到某个位置，该位置超出了当前最远位置，返回false
时间复杂度：O(n)，空间复杂度：O(1)
适用场景：贪心算法、数组操作、路径问题

30. 字符串消除游戏
题目：输入一个只包含英文字母的字符串，字符串中的两个字母如果相邻且相同，就可以消除。在字符串上反复执行消除的动作，直到无法继续消除为止，此时游戏结束。请输出最终得到的字符串长度。
输入描述：输入原始字符串str，只能包含大小写英文字母，字母大小写敏感，str长度不超过100。
输出描述：输出游戏结束后，最终得到的字符串长度。
示例1：输入 gg，输出 0（gg可以直接消除，得到空串）。
示例2：输入 mMbccbc，输出 3（先消除cc，得到mMbbc；再消除bb，得到mMc）。
算法：栈
- 使用栈来模拟消除过程
- 遍历字符串，对于每个字符：
  - 如果栈不为空且栈顶元素与当前字符相同，弹出栈顶
  - 否则，将当前字符压入栈
- 最终栈的大小就是剩余字符串的长度
时间复杂度：O(n)，空间复杂度：O(n)
适用场景：字符串处理、栈操作、模拟问题

31. 质因数分解

IPv6 地址由8组16进制的数字来表示，每组表示 16 比特。这些组数字通过 (":")分割。比如,  2001:0db8:85a3:0000:0000:8a2e:0370:7334 是一个有效的地址。而且，我们可以加入一些以 0 开头的数字，字母可以使用大写，也可以是小写。所以， 2001:db8:85a3:0:0:8A2E:0370:7334 也是一个有效的 IPv6 address地址 (即，忽略 0 开头，忽略大小写)。
然而，我们不能因为某个组的值为 0，而使用一个空的组，以至于出现 (::) 的情况。 比如， 2001:0db8:85a3::8A2e:0370:7334 是无效的 IPv6 地址。同时，在 IPv6 地址中，多余的 0 也是不被允许的。比如， 02001:0db8:85a3:0000:0000:8a2e:0370:7334 是无效的。
数据范围：字符串长度满足 5≤n≤50
示例1：输入："172.16.254.1"，返回值："IPv4"。
示例2：输入："2001:0db8:85a3:0:0:8A2E:0370:7334"，返回值："IPv6"。
示例3：输入："256.256.256.256"，返回值："Neither"。
算法：字符串处理
- 检查输入字符串是否包含"."或":"，判断是IPv4还是IPv6
- 对于IPv4：
  - 分割成4个部分
  - 每个部分必须是1-3位数字
  - 每个部分的值必须在0-255之间
  - 每个部分不能以0开头（除非是单独的0）
- 对于IPv6：
  - 分割成8个部分
  - 每个部分必须是1-4位16进制数字
  - 每个部分可以包含数字和字母（大小写均可）
时间复杂度：O(n)，空间复杂度：O(n)
适用场景：字符串处理、IP地址验证、网络编程

55. 阶乘末尾零的个数
题目：给定一个非负整数 n ，返回 n! 结果的末尾为 0 的数量。
n! 是指自然数 n! 的阶乘,即 : N!=1×2×3…(N−2)×(N−1)×N。特殊的,  0的阶乘是 1 。
数据范围： 0<n≤10的14次方
示例1：输入：3，返回值：0（3!=6）。
示例2：输入：5，返回值：1（5!=120）。
示例3：输入：1000000000，返回值：249999998。
算法：数学规律
- 末尾零的个数等于n中包含的5的因子个数
- 计算公式：count = n/5 + n/25 + n/125 + ...
时间复杂度：O(logn)，空间复杂度：O(1)
适用场景：数学问题、阶乘计算、因子分解

示例1：输入：[[10,30],[20,60],[80,100],[150,180]]，返回值：[[10,60],[80,100],[150,180]]。
示例2：输入：[[0,10],[10,20]]，返回值：[[0,20]]。
算法：排序+贪心
- 按区间起点排序
- 遍历排序后的区间，合并重叠的区间
- 维护当前合并区间的起点和终点
- 如果当前区间与合并区间重叠，更新终点
- 否则，添加合并区间到结果中，开始新的合并区间
时间复杂度：O(nlogn)，空间复杂度：O(n)
适用场景：区间操作、排序、贪心算法

例如：
给定二叉树 [3,9,20,null,null,15,7],

    3
   / \
  9  20
    /  \
   15   7

返回锯齿形层次遍历如下：
[  [3],  [20,9],  [15,7]]
算法：广度优先搜索
- 使用队列进行层次遍历
- 对于偶数层（从0开始），正序添加节点值
- 对于奇数层，逆序添加节点值
- 记录当前层数，决定是否反转当前层的节点值
时间复杂度：O(n)，空间复杂度：O(n)
适用场景：二叉树、层次遍历、广度优先搜索
